Terraform + Proxmox 快速参考卡片

最常用的 10 个操作
====================

1. terraform init          # 初始化项目
2. terraform validate      # 验证配置
3. terraform fmt -rec      # 格式化代码
4. terraform plan          # 查看计划
5. terraform apply         # 应用配置
6. terraform state list    # 列出资源
7. terraform state show    # 查看资源
8. terraform refresh       # 刷新状态
9. terraform output        # 查看输出
10. terraform destroy      # 销毁资源

最常见的 5 个问题与快速解决
============================

1. 磁盘缩容
   症状: VM 无法启动，显示 unused_disk
   解决: 确保 disk_size >= 模板大小

2. Output 不显示
   症状: terraform output 为空
   解决: 在 Root Module 中重新定义 output

3. VM 反复重建
   症状: plan 显示 will be destroyed
   解决: 检查 vmid 是否与 state 一致

4. QGA 无法运行
   症状: Terraform 超时卡住
   解决: 在模板中预装 qemu-guest-agent

5. Cloud-Init 未运行
   症状: SSH Key 未注入
   解决: 检查 cicustom 与其他参数的冲突

关键配置要点
============

磁盘配置:
  - 必须 disk_size >= template_size
  - type = "disk" （不是 scsi）
  - slot = 0
  - 指定存储池名称

网络配置:
  - ipconfig0 = "ip=dhcp"          （DHCP）
  - ipconfig0 = "ip=10.0.0.1/24"   （静态IP）

启动顺序:
  boot = "order=scsi0;ide2;net0"

生命周期规则:
  lifecycle {
    ignore_changes = [unused_disk, efidisk]
  }

UEFI 配置:
  bios = "ovmf"
  efidisk { efitype = "4m"; storage = "..." }
  cloudinit_cdrom_storage = "..."

Cloud-Init:
  ciuser = "ubuntu"
  sshkeys = file("~/.ssh/id_rsa.pub")
  agent = 1   # 启用 QEMU Guest Agent

SSH 密钥管理
============

原则:
  Terraform 建房子（注入 Bootstrap Key）
  Ansible 装修房子（日常维护 Key 列表）

流程:
  Day 0: Terraform 使用 Bootstrap Key 创建 VM
  Day 1: Ansible 部署 User Management Role
  Day 2+: 修改 Ansible 变量，运行 Playbook

状态管理
========

导入现有 VM:
  1. 定义资源框架
  2. terraform import proxmox_vm_qemu.name pve0/qemu/VMID
  3. terraform state show 查看属性
  4. 复制到代码，执行 plan

处理漂移:
  1. terraform plan 检查非预期变更
  2. 修改代码匹配实际状态 或 添加 ignore_changes
  3. 再次 plan 验证没有变更

常见错误消息
============

No permissions
  原因: Proxmox 用户权限不足
  解决: 检查 pveum 权限配置

QEMU guest agent is not running
  原因: VM 中 qemu-guest-agent 未启动
  解决: SSH 进入 VM，手动启动 systemctl start qemu-guest-agent

invalid vmid
  原因: VMID 被占用或格式错误
  解决: 使用 qm list 查看已有 VM ID

Could not find storage 'local-lvm'
  原因: 指定的存储池不存在
  解决: 检查 Proxmox 实际可用存储池（pvesm status）

clone failed
  原因: 模板不存在、权限问题、磁盘空间不足
  解决: 检查模板状态，查看 Proxmox 日志

Proxmox 常用命令
================

qm list                    # 列出所有 VM
qm config 100              # 查看 VM 100 配置
qm config 100 | grep disk  # 查看磁盘信息
qm resize 100 scsi0 +50G   # 扩展磁盘
pvesm status               # 查看存储池
pvesm list local-zfs       # 列出存储中的镜像
ls -la /var/lib/vz/snippets/  # 列出 Cloud-Init Snippet

调试技巧
========

启用详细日志:
  TF_LOG=DEBUG terraform plan
  TF_LOG=DEBUG TF_LOG_PATH=terraform.log terraform apply

查看资源状态:
  terraform state show 'proxmox_vm_qemu.web' | jq

查看执行计划的 JSON:
  terraform plan -json | jq '.resource_changes[]'

交互式 Terraform 求值:
  terraform console
  > var.vm_cores

强制解锁（谨慎）:
  terraform force-unlock LOCK_ID

记住
====

最重要的一点：
  在执行任何破坏性操作前，务必仔细阅读 terraform plan 的输出！

黄金法则：
  显式优于隐式
  不要依赖默认值，显式定义所有关键参数

防御性编程：
  在生成 Terraform 变量的脚本中加入充分的验证和单位转换
