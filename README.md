# Infrastructure as Code (IaC)

This repository contains Infrastructure as Code for managing a Proxmox VE cluster using Ansible and Terraform.

## Project Overview
- **Terraform**: VM provisioning on Proxmox and Netbox resource management
- **Ansible**: Application configuration management (Docker, Netbox, Samba, Immich)
- **Netbox**: IPAM and DCIM source of truth

### Separation of Concerns
- **Provisioning (Terraform)**: Creates and manages VM infrastructure (CPU, memory, disk, network)
- **Configuration (Ansible)**: Installs and configures applications on provisioned VMs

## Environment Setup

To ensure a consistent environment, this project uses a local Python virtual environment.

1.  **Initialize the environment**:
    ```bash
    ./scripts/setup_env.sh
    ```

2.  **Activate the environment**:
    ```bash
    source .venv/bin/activate
    ```

## Version Control & Security
To maintain a clean and secure repository, the following files are excluded via `.gitignore`:

- **Terraform State (`*.tfstate`, `*.tfstate.backup`)**: Contains sensitive infrastructure data (IPs, IDs, potential secrets). **Never commit these.** In a team environment, use a remote backend (e.g., S3, Consul).
- **Terraform Variables (`*.tfvars`)**: Contains sensitive credentials (passwords, API tokens). Use `terraform.tfvars.example` as a template instead.
- **Terraform Plugins (`.terraform/`)**: Large, platform-specific provider binaries downloaded during `terraform init`.
- **Virtual Environment (`.venv/`)**: Local Python dependencies.
- **Agent Artifacts (`.agent/`)**: Temporary files generated by AI agents.
- **Python Cache (`__pycache__/`, `*.pyc`)**: Compiled Python bytecode.

## Quick Start

### 1. Infrastructure Provisioning
**Terraform**
Used for all infrastructure (Netbox, Samba, Immich).
```bash
cd terraform/proxmox
terraform init && terraform apply
```

### 2. Configuration Management
Used to configure applications on provisioned VMs.
```bash
cd ansible/

# Deploy Applications
ansible-playbook playbooks/deploy-netbox.yml
ansible-playbook playbooks/deploy-samba.yml
ansible-playbook playbooks/deploy-immich.yml
```

### 3. Netbox Configuration
Manage Netbox internal resources (Sites, VMs, IPs) via Terraform.
```bash
cd terraform/netbox
terraform init && terraform apply
```

### Utility Operations
```bash
cd ansible/
ansible proxmox_cluster -m ping
```

## Troubleshooting

### Terraform Provider Crash & Version Compatibility

**Symptoms:**
*   `terraform plan` or `terraform import` crashes with `panic: interface conversion: interface {} is nil, not string`.
*   Permission errors like `missing: [VM.Monitor]` on Proxmox VE 8+.

**Cause:**
*   **v3.0.2-rc05**: Contains a regression causing crashes when parsing certain VM configurations (e.g., existing snapshots, specific disk settings).
*   **v2.9.14**: Incompatible with Proxmox VE 8+ privilege changes (expects `VM.Monitor` permission which was removed/changed).

**Solution:**
1.  **Use Provider Version `3.0.2-rc04`**: This version bridges the gap, supporting PVE 8 permissions while avoiding the rc05 crash regression.
    ```hcl
    terraform {
      required_providers {
        proxmox = {
          source  = "telmate/proxmox"
          version = "3.0.2-rc04"
        }
      }
    }
    ```
2.  **Use API Token Authentication**: Password authentication for `root@pam` can fail permission checks in older/specific provider versions. Use an API Token with `Administrator` privileges (or disable privilege separation for the token).
    *   Generate token: `pveum user token add root@pam terraform --privsep 0`
    *   Configure in `provider.tf` using `pm_api_token_id` and `pm_api_token_secret`.

### General Troubleshooting
*   **Clear Cache**: `rm -rf .terraform .terraform.lock.hcl && terraform init -upgrade`
